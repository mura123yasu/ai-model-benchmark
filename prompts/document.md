# 文書プロンプト集

---

## 1. 文書作成

### プロンプト

```
以下の条件でプレスリリースを作成してください。

【企業情報】
- 会社名：TechNova株式会社
- 事業内容：AIを活用したビジネスソリューションの開発・提供

【発表内容】
- 製品名：StockAI
- 製品カテゴリ：AI在庫管理SaaS
- 特徴：AIが需要予測を行い、在庫過不足を自動最適化する

【出力形式】
- 文字数：400〜500字（タイトルと会社概要を含む）
- 構成：タイトル / リード文 / 本文 / 会社概要
- 文体：ビジネス向けプレスリリース形式

上記の条件をすべて満たしたプレスリリースを出力してください。
```

### 評価ポイント

- **構成の正確さ**：タイトル・リード文・本文・会社概要の4要素がすべて揃っているか
- **ビジネス文体**：プレスリリースとして自然な敬語・表現が使われているか
- **文字数遵守**：400〜500字の範囲に収まっているか（タイトル・会社概要を含む）

## 2. 文書レビュー

### プロンプト

```
以下のビジネスメールをレビューしてください。

【レビュー対象メール】
---
件名：ご確認のお願い

山田様

お疲れ様です。鈴木です。
先日の件ですが、どうなりましたでしょうか。
なるべく早めにご返答いただけると幸いです。
資料は添付しておきます。
よろしくです。

鈴木
---

【出力形式】
以下の4カテゴリに分けて問題点を指摘し、それぞれ改善案を提示してください。

1. 敬語・表現
2. 構成
3. 内容の明確さ
4. その他

各カテゴリで問題点がなければ「問題なし」と記載してください。
```

### 評価ポイント

- **問題点の網羅性**：以下の問題をすべて指摘できているか
  - 「お疲れ様です」は社外メールでは不適切
  - 「先日の件」が何を指すか不明瞭
  - 「どうなりましたでしょうか」は二重敬語
  - 「なるべく早めに」では期限が不明確
  - 「よろしくです」は失礼な表現
  - 件名が内容を表していない
  - 添付資料の説明がない
- **改善案の具体性**：抽象的な指摘ではなく、具体的な修正文例が示されているか

## 3. 要約

### プロンプト

```
以下の文章を3つのパターンで要約してください。

【要約対象文章】
---
デジタルトランスフォーメーション（DX）とは、企業がデジタル技術を活用してビジネスモデルや業務プロセスを根本から変革し、競争優位性を確立する取り組みのことです。単なるIT化とは異なり、組織文化や働き方そのものを変えることが求められます。日本では2025年の崖問題として、レガシーシステムの刷新が急務とされており、経済産業省も積極的に推進しています。しかし中小企業では人材・予算不足から導入が遅れており、大企業との格差が広がっています。
---

【出力形式】
パターン1：1文で要約（30字以内）
パターン2：箇条書き3点で要約
パターン3：100字程度の文章で要約

各パターンのラベルを明記し、制約を厳守してください。
```

### 評価ポイント

- **制約遵守**：パターン1は30字以内、パターン2は3点のみ、パターン3は100字程度に収まっているか
- **情報の取捨選択の質**：制約の中で最も重要な情報（DXの定義・日本の現状・格差）を適切に優先しているか
